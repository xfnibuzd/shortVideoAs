- **版本**：v1.1（基于《需求大概.docx》落地，已纳入需求确认结论）
- **状态**：需求已确认，待技术评审
- **文档目标**：将原始需求草稿与 3 张界面草图，整理为研发可直接拆任务、可验收的产品需求文档。

## 1. 产品概述

### 1.1 背景与定位
面向短剧/短视频创作者的 AI 辅助创作工具。用户以「项目」为单位组织一部作品，在工作台中按「章节 → 分镜」拆解剧本，并依据剧本内容、资产（人物、背景、道具）与提示词模版调用 AI 生成分镜图片，最终用于后续成片。

### 1.2 核心用户流程
```
创建项目 → 进入工作台 → 维护章节/分镜 → 编写分镜剧本
        → 维护资产(人物/背景/道具) → 配置/选择提示词模版
        → 生成分镜图片(选资产+模版→提交AI→监控进度) → 查看结果
```

### 1.3 名词定义
- **项目（Project）**：一部作品，包含章节、分镜、资产、生成产物。
- **章节（Chapter）**：剧本的组织单元，下挂多个分镜。
- **分镜（Shot/Scene）**：章节下的最小创作单元，含剧本文本与分镜图片。
- **资产（Asset）**：项目内可复用的创作素材，分人物、背景、道具三类（不跨项目共享）。
- **分镜图片**：由 AI 依据剧本+资产+提示词模版生成的图片；**生成内容与图片数量由提示词控制**（原草稿「九宫格」为表述误差，非固定 9 张）。
- **提示词模版（Prompt Template）**：项目级可复用的生成提示词，供图片生成调用。

## 3. 详细需求

### M1 · 项目列表页（P0）

**界面参考**：image1（顶部 Tab + 卡片网格 + 右上角「创建项目」）

**功能点**
- 顶部 Tab：`我的项目(数量)`；`我参与的项目` 因协作不在首期范围，**首期仅展示「我的项目」**（草图中的「我参与的项目」Tab 可隐藏或置灰占位）。
- 项目以卡片网格展示，每张卡片包含：
  - 封面图（无封面时显示占位图）
  - 项目名称
  - 创建时间（格式 `YYYY-MM-DD HH:mm:ss`）
  - 章节数标签（如「共 5 章」，右下角覆盖在封面上）
  - 右上角「更多」菜单（`···`：重命名、删除等）
  - ~~协作人数~~（协作不纳入首期范围，首期不展示）
- 右上角「+ 创建项目」按钮，点击打开创建项目弹窗（M2）。
- 点击卡片主体进入工作台（M3）。
- 空状态：无项目时展示引导与「创建项目」按钮。

**验收标准**
- [ ] Tab 数量与实际项目数一致。
- [ ] 卡片信息字段齐全且时间格式正确。
- [ ] 点击卡片进入对应项目工作台。
- [ ] 「更多」菜单支持重命名、删除（删除需二次确认）。

### M2 · 创建项目弹窗（P0）

**界面参考**：image2

> ✅ **需求结论**：MVP **仅需输入项目名称**。草图 image2 中的「叙事模式 / 创作模式 / 剧本」等字段**不纳入首期**，作为后续迭代储备。

**MVP 字段（P0，首期唯一字段）**
- `项目名称`（必填，文本输入，限长建议 50 字）。
- 操作按钮：`取消`、`立即创建`。

**后续迭代储备字段（不在首期范围）**
- `叙事模式`：`剧情模式` / `旁白解说模式`。
- `创作模式`：`生图转视频` / `参考生视频`。
- `剧本/画面提示词`：多行文本，支持「AI 随机生成剧本」。

**功能点**
- 项目名称为空时「立即创建」按钮禁用。
- 创建成功后关闭弹窗，刷新列表并默认进入新项目工作台。

**验收标准**
- [ ] 仅填名称即可成功创建项目。
- [ ] 名称为空无法提交。
- [ ] 创建后列表实时更新。

### M3 · 工作台页（P0）

**界面参考**：image3。三栏布局：**左（章节/分镜目录）｜中（内容编辑区）｜右（资产/全局设定）**。

#### M3.1 顶部栏
- 左：`返回` + 项目名（可编辑 ✎）。
- 中：页面级 Tab **仅保留两个**：`剧本`、`分镜图片`（草图中的 `概览/主体/分镜/故事板/成片` 不纳入首期）。
  - 该两个 Tab 可能随当前选中分镜切换内容（与 M3.3 中栏内容区为同一组 Tab）。
- 右：分享等操作（P1，选能）。

#### M3.2 左栏 — 章节/分镜目录
- 树形结构：`章节` 下挂多个 `分镜`。
- 支持：新增章节（「+ 添加剧集/章节」）、新增分镜、重命名、删除、排序（拖拽，P1）。
- 点击某个分镜 → 中间内容区切换为该分镜内容。
- 当前选中项高亮。

#### M3.3 中栏 — 内容编辑区
- 顶部两个 Tab：`剧本`、`分镜图片`（即 M3.1 顶部保留的两个 Tab）。
- **剧本 Tab**：
  - 文本编辑区，展示/编辑当前分镜剧本。
  - 空状态：无内容时为可输入的空态。
  - **自动保存**：输入后自动保存（建议防抖 1–2s，含保存状态提示）。
  - 字数统计（草图示 `0/2000`）。
- **分镜图片 Tab**：展示该分镜的图片生成结果，逻辑见 M4。

#### M3.4 右栏 — 资产区 / 全局设定
- **资产**：分三类 Tab/分组——`人物`、`背景`、`道具`。
  - 每类支持：新增、编辑、删除资产；资产含名称、图片/描述等。
  - 资产为项目内复用（不跨项目共享），在分镜图片生成时被选择引用。
- **全局设定（草图 image3，P1）**：
  - `视频比例`：`16:9` / `9:16` / `4:3` / `3:4`。
  - `创作模式`：`生图转视频` / `参考生视频`。
  - `风格参考`：`写实` / `动漫`，下挂风格预设网格（古风写实、真人写实…）。

**验收标准**
- [ ] 左栏可创建多个分镜，点击切换中栏内容。
- [ ] 中栏剧本输入后自动保存且刷新不丢失。
- [ ] 中栏可在「剧本/分镜图片」间切换。
- [ ] 右栏可维护人物/背景/道具三类资产。

### M4 · 分镜图片生成（P0）

**入口**：工作台「分镜图片」Tab。

**功能点**
- 展示该分镜过往生成的图片；无记录时仅展示「+ 新增图片」按钮。
- 点击「新增/生成」→ 打开生成配置弹窗：
  - 选择 `角色`（人物，**可多选**）。
  - 选择 `道具`。
  - 选择 `背景`。
  - 选择 `提示词模版`（来自 M5，项目级）。
- 确认后：系统组装（剧本内容 + 所选资产 + 提示词模版）→ 提交 AI 生成图片。
- **生成内容与图片数量由提示词控制**（不是固定 9 张，原草稿「九宫格」为表述误差）。
- **进度监控与状态维护**：
  - 生成为异步任务，需维护任务状态：`排队中 / 生成中(含进度%) / 成功 / 失败`。
  - **用户切换到其他页面/分镜后再回来，进度与状态仍准确**（状态持久化，不依赖页面停留）。
  - 失败可重试，成功后展示生成结果。

**验收标准**
- [ ] 无历史记录时仅显示新增按钮。
- [ ] 生成弹窗支持多选角色，单选/多选道具、背景与模版。
- [ ] 生成图片数量与内容随提示词变化。
- [ ] 提交后展示进度，离开页面再返回状态不丢失。
- [ ] 生成成功展示图片，失败可重试。

### M5 · 提示词模版维护（项目级，P1）

**作用域**：提示词模版为**项目级**，仅在当前项目内可见与复用，不跨项目共享。

**功能点**
- 模版列表：展示当前项目已保存的提示词模版（名称 + 内容预览）。
- 「+ 新增」按钮：创建模版（名称 + 提示词内容）。
- 支持编辑、删除。
- 新增/已有模版供 M4 图片生成选择调用。

**验收标准**
- [ ] 可新增、编辑、删除模版。
- [ ] 模版仅在所属项目内可见（不跨项目）。
- [ ] 新建模版即时可在图片生成弹窗中选择。

## 4. 数据模型（建议）

```
Project        { id, name, cover, createdAt, chapterCount }
Chapter        { id, projectId, title, order }
Shot           { id, chapterId, title, order, scriptContent, updatedAt }
Asset          { id, projectId, type(person|background|prop), name, image, desc }
PromptTemplate { id, projectId, name, content, createdAt }   // 项目级
Generation     { id, shotId, assetIds[], templateId, status(queued|running|
                 success|failed), progress, resultImages[], createdAt }
```

## 5. 关键接口（建议）

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/projects?type=mine` | 项目列表（首期仅 mine） |
| POST | `/projects` | 创建项目（仅 name） |
| PATCH/DELETE | `/projects/{id}` | 重命名/删除 |
| GET/POST | `/projects/{id}/chapters` | 章节列表/新增 |
| GET/POST | `/chapters/{id}/shots` | 分镜列表/新增 |
| PUT | `/shots/{id}` | 保存分镜剧本（自动保存） |
| CRUD | `/projects/{id}/assets` | 资产维护（项目内） |
| CRUD | `/projects/{id}/prompt-templates` | 提示词模版（项目级） |
| POST | `/shots/{id}/generations` | 提交分镜图片生成 |
| GET | `/generations/{id}` | 轮询/查询生成状态与进度 |

## 6. 状态与异常

- **自动保存**：防抖触发，展示「保存中/已保存/保存失败重试」。
- **生成任务**：状态持久化于服务端，前端轮询或 WebSocket 推送进度；切页返回需恢复状态。
- **空状态**：项目、章节、分镜、分镜图片、模版均需空态引导。
- **失败兜底**：生成失败提示原因并支持重试；网络异常友好提示。

| # | 问题 | 结论 |

## 8. 里程碑建议

- **M1（MVP）**：项目列表(仅我的项目) + 创建项目(仅名称) + 工作台(章节/分镜/剧本自动保存/资产) + 分镜图片生成与进度。
- **M2**：提示词模版维护(项目级) + 全局设定(比例/风格) + 创建项目储备字段。
- **M3**：协作、其余 Tab(概览/主体/故事板/成片)、排序拖拽等增强。
